# Use private S3-compatible storage with authenticated short-lived downloads

The optional S3-compatible file-storage driver uses AWS SDK v3 and keeps buckets private. Production buckets are provisioned outside the application, while explicitly configured development and test environments may create their bucket; after backend authentication and authorization, S3-backed downloads redirect to a 60-second presigned URL so object transfer does not consume API service bandwidth. Local filesystem storage remains the default, and the pinned official `rustfs/rustfs:1.0.0-beta.8` image provides the optional Docker-backed S3 compatibility test environment. RustFS is a test backend rather than an application dependency or selected production provider, and the application must not use RustFS-specific APIs. The configured active driver controls new uploads only; each Managed File retains its storage location, and reads, previews, and deletes continue through that recorded driver without automatically migrating historical content. Its Object Location persists the storage driver, optional bucket, and complete object key; endpoints, credentials, and temporary download URLs remain runtime concerns rather than file metadata.

Logical deletion invalidates the Managed File and its File References before physical content is removed. A worker performs Content Deletion asynchronously and records its completion so external storage failures can be retried without exposing invalid files or repeatedly deleting content; uploads that reach storage but fail metadata persistence use best-effort compensating deletion.

Runtime configuration uses `FILE_STORAGE_DRIVER=local|s3`, with local remaining the default. S3 configuration requires a region and bucket, permits an optional endpoint and normalized object prefix, supports path-style requests and explicitly enabled development/test bucket creation, and accepts either a complete explicit credential set or the AWS SDK default credential chain. API and worker processes share the same validated configuration contract; the presigned URL lifetime defaults to 60 seconds and is constrained to 15-900 seconds.

Normal tests and push CI use adapter mocks and contract tests without requiring an object-storage service. Real S3 compatibility is verified through `pnpm test:s3-integration` and a manually triggered workflow against the pinned RustFS image, preventing an optional external image or registry outage from blocking ordinary pushes while keeping the compatibility check reproducible.
